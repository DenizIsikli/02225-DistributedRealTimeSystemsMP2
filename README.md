# TSN Credit-Based Shaper Analysis: CBS vs Strict Priority

This project implements worst-case delay (WCD) analysis, discrete-event simulation, and comparison of **Credit-Based Shaper (CBS)** and **Strict Priority (SP)** scheduling for periodic real-time streams in a **Time-Sensitive Networking (TSN)** context.

## 🚀 Running the Analysis

All scripts are run as Python modules from the **project root directory**.

### CBS Analytical WCRT

Computes worst-case response times analytically using the CBS eligible-interval method (Cao et al., RTNS'16).

```bash
py -m src.run_analysis test-case-1
```

Optional arguments:
```
--idle-slope 0.5     idleSlope fraction (α⁺), default 0.5
--send-slope 0.5     sendSlope fraction (α⁻), default 0.5
--verbose            Print per-link breakdown (SPI, HPI, LPI)
```

### Discrete-Event Simulator (CBS)

Simulates TSN streams with CBS shaping over multiple hyperperiods.

```bash
py -m src.run_simulator test-case-1 --hyperperiods 500
```

Optional arguments:
```
--hyperperiods 200   Number of hyperperiods to simulate, default 200
--duration <µs>      Explicit simulation duration in µs (overrides hyperperiods)
--idle-slope 0.5
--send-slope 0.5
```

### Analytical vs. Simulated Comparison (CBS)

Runs both analysis and simulation and compares analytical upper bounds against observed simulation results.

```bash
py -m src.run_comparison test-case-1 --hyperperiods 500
```

### CBS vs. Strict Priority Comparison

Runs both CBS and SP analysis and simulation side by side. Demonstrates the credit mechanism's protection of lower-priority queues against starvation.

```bash
py -m src.run_sp_comparison test-case-1 --hyperperiods 500
```

### How to Interpret the Output

**`run_analysis.py`** prints a table like:

```
  ID  Class   Size   Period  Deadline       WCRT        Ref  Match  Sched
   0      A   1012     1000      1000     603.20     603.20      ✓    yes
```
- `WCRT` — Computed worst-case response time in µs
- `Ref` — Reference value from `*-WCRTs.csv` (if present)
- `Match ✓/✗` — Whether computed value matches reference (tolerance 0.01 µs)
- `Sched yes/NO` — Whether `WCRT` ≤ `Deadline`

**`run_simulator.py`** prints observed response times:

```
  ID  Class   Size   Period  Frames      MinRT      AvgRT      MaxRT  Deadline
   0      A   1012     1000     400     161.92     273.24     385.68      1000
```
- `Frames` — Total number of frames delivered during simulation
- `MinRT / AvgRT / MaxRT` — Minimum, average, maximum observed response time in µs

**`run_comparison.py`** shows the pessimism gap between analysis and simulation:

```
  ID  Class  Analytical    Sim Max    Gap%  Bound
   0      A      603.20     385.68   36.1%      ✓
```
- `Gap%` — Percentage by which the analytical bound exceeds the simulation maximum
- `Bound ✓/✗` — Whether the analytical WCRT is a valid upper bound on the simulated max

**`run_sp_comparison.py`** compares CBS and SP side by side:

```
  ID  Cls   CBS_Ana   CBS_Sim    SP_Ana    SP_Sim  CBS_OK   SP_OK
   4    B    884.48    212.00    705.60    278.16     yes     yes
```
- Class B streams under SP have higher simulated response times, demonstrating starvation risk without CBS

## 🧠 Algorithm Descriptions

### Credit-Based Shaper (CBS)

CBS is the AVB traffic shaper defined in IEEE 802.1Qav used in TSN to bound interference between priority classes.

**Key Properties:**
- Each queue has a credit counter that grows while waiting and decreases while transmitting
- A frame may only be sent when credit ≥ 0
- Bandwidth is reserved via `idleSlope` (α⁺) and `sendSlope` (α⁻) parameters
- Prevents starvation of lower-priority queues by bounding high-priority bursts

**WCRT formula (Cao et al., RTNS'16, Theorem 3):**

```
WCRT_i = Σ_{l ∈ path_i} (C_i + SPI_i^l + HPI_i^l + LPI_i^l)

SPI_i^l = Σ_{j ≠ i, P_j = P_i} C_j × (1 + α⁻/α⁺)   (same-priority interference)
LPI_i^l = max_{P_j < P_i} C_j                          (lower-priority blocking)

Class A (highest): HPI = 0
Class B:           HPI = LPI × (α⁺_H / α⁻_H) + max_{P_j > P_i} C_j
```

### Strict Priority (SP)

Standard fixed-priority non-preemptive scheduling applied per output port.

**Key Properties:**
- Higher-priority frames always preempt lower-priority transmissions
- No credit mechanism — high-priority bursts can monopolize the link
- Classical Response Time Analysis (RTA) with fixed-point iteration is used

**WCRT formula (per link, fixed-point iteration):**

```
R_i^(0) = C_i
R_i^(n+1) = C_i + B_i + SPI_i + Σ_{j: P_j > P_i} ⌈R_i^(n) / T_j⌉ × C_j

B_i   = max_{P_j < P_i} C_j        (non-preemptive blocking)
SPI_i = Σ_{P_j = P_i, j ≠ i} C_j  (same-priority, one frame each)
```

Converges when `R_i^(n+1) = R_i^(n)`, or is declared unschedulable if `R_i > D_i`.

## 📖 Test Case File Format

Each test case folder contains:

| File | Description |
|---|---|
| `*-topology.json` | Network nodes (end systems, switches) and link speeds |
| `*-streams.json` | Stream parameters: size, period, deadline, priority class (PCP) |
| `*-routes.json` | Directional routing paths for each stream across the topology |
| `*-WCRTs.csv` | Reference WCRT values for validation (optional) |

**Priority classes (PCP values):**
- `2` — Class A (highest AVB priority)
- `1` — Class B (lower AVB priority)
- `0` — Best Effort (BE, no WCRT guarantee)

---

## 🔧 Stream Model

Each stream τ_i is characterized by:

| Parameter | Description |
|---|---|
| `id` | Unique stream identifier |
| `size` | Maximum frame payload size (bytes) |
| `period` | Minimum inter-arrival time (µs) |
| `deadline` | Maximum allowed end-to-end delay (µs) |
| `pcp` | Priority class (0 = BE, 1 = Class B, 2 = Class A) |

Streams traverse multiple hops along a defined route. The end-to-end WCRT is the sum of per-link WCRTs (compositional analysis).
